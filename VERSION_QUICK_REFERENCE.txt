# Version Management Quick Reference Card

## 🎯 Current Version: 3.0.1 (Oct 28, 2025)

## ⚡ Quick Bump Decision

```
❓ What did you change?

🔧 Bug fix or small tweak?
   → Update: Z (Patch) → 3.0.2
   
✨ New feature or capability?
   → Update: Y (Minor) → 3.1.0
   
💥 Breaking change or redesign?
   → Update: X (Major) → 4.0.0
```

---

## 📝 Files to Update (Every Time)

```
1. esp32-scheduler.ino          (Line 3-6)

```

## 🚀 Commit Template

```bash
git commit -m "vX.Y.Z: [type] Short description

- Change 1
- Change 2
- Change 3"
```

**Types**: `[patch]` `[minor]` `[major]`

## 📊 Example Versions

```
v3.0.1 - Bug fix or small improvement
v3.1.0 - New feature added
v4.0.0 - Breaking change / major redesign
```

## 🔍 Date Format

**Always use**: `Month DD, YYYY`

❌ Wrong: `10-28-2025` or `2025-10-28`  
✅ Right: `October 28, 2025`
